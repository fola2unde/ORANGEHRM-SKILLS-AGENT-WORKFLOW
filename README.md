# OrangeHRM Skills Agent Workflow

This repository contains a Playwright-based QA automation framework for the OrangeHRM public demo application. It was built as a foundation for skills-and-agents-driven test automation work, with page objects, reusable fixtures, accessibility checks, HTTP contract checks, CI configuration, and documentation.

## Application Under Test

The framework targets the OrangeHRM demo site:

```text
https://opensource-demo.orangehrmlive.com
```

Primary login route:

```text
https://opensource-demo.orangehrmlive.com/web/index.php/auth/login
```

## Framework Capabilities

- TypeScript Playwright test framework
- Page Object Model for Login, Dashboard, and PIM pages
- Custom fixtures for page objects and authenticated sessions
- Centralized test data and route constants
- Smoke, regression, accessibility, and HTTP contract test tags
- HTML, JSON, JUnit, screenshots, videos, and trace artifacts
- Cross-browser and responsive project configuration
- GitHub Actions workflow for CI execution

## Prerequisites

- Node.js 18 or later
- npm
- Playwright browser dependencies

## Setup

From the repository root:

```bash
cd framework
npm install
npx playwright install
```

Optional local environment file:

```bash
copy .env.example .env
```

The default `BASE_URL` is already configured for OrangeHRM:

```text
https://opensource-demo.orangehrmlive.com
```

## Test Suites

### Authentication

Located in `framework/tests/e2e/auth.spec.ts`.

Coverage includes valid login, invalid login, logout, required username validation, password masking, and forgot password navigation.

### Dashboard

Located in `framework/tests/e2e/dashboard.spec.ts`.

Coverage includes dashboard load, core widgets, user menu, quick launch controls, and navigation into Leave flows.

### PIM

Located in `framework/tests/e2e/pim.spec.ts`.

Coverage includes employee list load, record summary, employee ID search, filter reset, and Add Employee navigation.

### Accessibility

Located in `framework/tests/accessibility/wcag-compliance.spec.ts`.

Coverage includes axe scans, labels, color contrast, keyboard navigation, headings, link text, landmarks, form validation, and focus behavior.

### HTTP Contracts

Located in `framework/tests/api/user-api.spec.ts`.

Coverage includes login page availability, protected route redirects, response headers, invalid route behavior, and response time.

## Reports And Artifacts

Playwright writes generated artifacts under:

```text
framework/test-results/
framework/playwright-report/
framework/blob-report/
```

These directories are ignored by Git and can be regenerated by running the tests.

## CI/CD

The framework includes a GitHub Actions workflow at:

```text
framework/.github/workflows/tests.yml
```

The workflow installs dependencies, installs Playwright browsers, runs browser-specific suites, runs accessibility and API tests, and uploads reports.

## Notes

- The OrangeHRM demo site is a live public environment, so data and timing can vary.
- Headed full-suite runs cover all configured projects and can take several minutes.
- For faster local feedback, prefer Chromium-only smoke runs before executing the full cross-browser matrix.
